Архиватор (ARCH)

это процесс, который копирует онлайновые файлы журнала повторения в архивную память, когда они переполняются.

Буфер журнала повторения

это циклический буфер в SGA, который содержит информацию об изменениях, выполненных в базе данных

Восстановитель (RECO)

это процесс, используемый с распределенной опцией ORACLE, которая автоматически исправляет сбои, в которых участвуют распределенные транзакции. Восстановитель используется для разрешения распределенных транзакций, зависших в результате сетевого или системного сбоя в распределенной базе данных.

Временный сегмент

временное рабочее дисковое пространство для промежуточных этапов обработки предложения SQL

Выделенный серверный процесс

это отдельный серверный процесс, созданный от имени каждого пользовательского процесса (или теневым процессом), потому что этот серверный процесс действует лишь от имени ассоциированного пользовательского процесса.

Глобальная область программы (PGA)

это область памяти, содержащая данные и управляющую информацию для одного процесса (серверного или фонового).

Глобальная область системы (SGA)

это группа структур разделяемой памяти, распределяемой ORACLE, которая поддерживает данные и управляющую информацию для одной инстанции ORACLE.

это область разделяемой памяти; все пользователи, присоединенные к многопользовательской инстанции базы данных, могут использовать информацию в SGA этой инстанции.

Драйверы

это компоненты программного обеспечения, которые занимаются транспортировкой данных, обычно через сеть.

Инстанция базы данных ORACLE

это совокупность SGA и процессов ORACLE.

Контрольная точка (CKPT)

в специфические моменты времени все модифицированные буфера в SGA записываются процессом DBWR в файлы данных; это событие называется контрольной точкой.

Личная область SQL

это область памяти, содержащая такие данные, как информация привязки и буфера времени выполнения.

Монитор процессов (PMON)

выполняет восстановление процесса, когда пользовательский процесс сбивается. PMON отвечает за очистку кэша и освобождение ресурсов, использовавшихся процессом.

Области программных кодов

это участки памяти, используемые для хранения кода, который выполняется или может быть выполнен.

Параметр PCTFREE

устанавливает процент памяти блока, резервируемой (оставляемой свободной) для возможных обновлений строк, уже содержащихся в блоке.

Писатель базы данных (DBWR)

записывает модифицированные блоки из буферного кэша базы данных в файлы данных.

Пользовательский процесс

создается и поддерживается для исполнения программного кода прикладной программы (такой как программа Pro*C) или инструмента ORACLE (такого как SQL*DBA). Пользовательский процесс также управляет взаимодействием с процессами сервера

Программный интерфейс

это слой программного обеспечения между приложением базы данных и ORACLE.

Процесс

это "канал управления", или механизм в операционной системе, способный выполнять последовательность шагов. Некоторые операционные системы используют термины "задание" или "задача".

Процесс монитора системы (SMON)

выполняет восстановление инстанции при запуске инстанции. SMON также отвечает за очистку временных сегментов, когда они больше не используются; он также сжимает непрерывные свободные экстенты, чтобы сделать доступным большее количество блоков свободной памяти. В среде Параллельного сервера SMON осуществляет восстановление инстанции для сбившегося процессора или инстанции.

Разделяемая область SQL

это область памяти, которая содержит дерево разбора и план исполнения для одного предложения SQL.

Разделяемый пул

это область в SGA, содержащая такие конструкты, как разделяемые области SQL и кэш словаря данных.

Размер табличного пространства

это размер его файла данных или суммарный размер всех файлов данных, составляющих это табличное пространство

Сбой процесса

это ошибка в пользовательском процессе, обращающемся к ORACLE, например, аварийное разъединение или прекращение процесса.

Связь баз данных

это именованный объект, который описывает "путь" от одной базы данных к другой.

Сегмент

это совокупность экстентов, распределенных для специфического типа структуры данных, и находящихся в одном и том же табличном пространстве.

Сегмент отката

это часть базы данных, в которой записываются действия транзакций, которые, возможно, придется отменить в некоторых обстоятельствах

Сессия

это конкретное соединение пользователя с инстанцией ORACLE через пользовательский процесс

Соединение

это коммуникационный путь между пользовательским процессом и инстанцией ORACLE

Файлы данных

это физические структуры, каждая из которых связана с одним табличным пространством

Экстент

это специфическое число смежных блоков данных, распределяемых для хранения специфического типа информации.
